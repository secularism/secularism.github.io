# Tailwind 版个人主页重构设计

**日期：** 2026-04-25

## 背景

当前主页已经具备单页结构、主题切换和基础交互，但整体视觉偏重，存在“稍微有点花里胡哨”的问题。用户希望将站点重构为更朴素、克制、适合作为 `github.io` 个人主页长期维护的版本，并明确要求：

- 标签页标题改为 `secularism`
- 替换默认 favicon，不使用旧的 Vue 默认图标
- 使用 Tailwind CSS 进行样式重构
- 使用可控的 UI 组件方案替换当前部分元素
- 对不同屏幕尺寸进行适配
- 保留并增强轻量动画效果，如 hover、滚动出现、背景变化等

## 目标

将当前首页重构为一个更简洁、朴素、现代的单页个人主页：

- 保持单页结构与锚点导航
- 降低视觉噪声，减少强玻璃感、重阴影和炫光
- 使用 Tailwind CSS 提升样式组织能力与响应式开发效率
- 以“可控基础组件 + 统一视觉”的方式替换当前元素
- 保留后续继续添加动画的扩展空间

## 风格方向

### 视觉关键词

- 简单
- 克制
- 现代
- 工程感
- 稳定
- 有轻微动态但不花哨

### 风格定位

页面整体位于“开发者文档风”和“作品集留白风”的中间地带：

- 不做赛博风、霓虹风、重发光风格
- 不做明显模板站风格
- 以留白、边框、层次和排版建立高级感
- 以轻量动态增强可读性和细节体验

## 信息架构

仍然保持单页结构，不拆分多页面，包含以下区块：

1. `Hero`
   - 站点标题 / 名字
   - 一句话定位
   - 两个主要操作按钮
   - 顶部导航与主题切换

2. `About`
   - 简短自我介绍
   - 个人标签
   - 一段带个人风格的描述

3. `Projects`
   - 3 个精选项目
   - 更简洁的项目卡片结构

4. `Stack`
   - 分组展示技术方向
   - 不采用密集技能墙

5. `Contact`
   - 联系方式和外链
   - 页面收尾 CTA

## 技术方案

## 技术栈

- Vite
- Tailwind CSS
- 原生 ESM 或轻量 JS 模块
- GitHub Pages 静态部署

## UI 组件方案

采用“Tailwind + 可控基础组件”的混合策略，而不是重度引入强视觉组件库。

实现原则：

- 用 Tailwind 构建整体布局、颜色、间距和响应式规则
- 用少量基础组件承载常用元素
- 不直接套用有明显“组件库味道”的默认主题

建议重构为以下基础组件：

- `SiteHeader`
- `ThemeToggle`
- `PrimaryButton / SecondaryButton`
- `SectionTitle`
- `ProjectCard`
- `InfoCard`
- `ContactLink`

这些组件可以是语义化 HTML + 可复用 Tailwind class 组合，不必引入复杂运行时框架。

## 为什么选择 Tailwind

- 便于快速重构视觉系统
- 更适合做响应式分层
- 更容易统一 hover、focus、motion 状态
- 比当前手写大体量 CSS 更便于持续维护

## 视觉系统

### 色彩

不再使用高对比炫光式视觉，改为中性配色：

- 浅色主题：白、浅灰、冷灰蓝
- 深色主题：深灰、煤黑、低饱和蓝灰
- 强调色：极少量低饱和蓝色，用于交互和焦点

### 背景

背景效果保留，但大幅降噪：

- 去掉明显“发光团块”视觉
- 使用轻微背景渐变或很弱的纹理层
- 滚动时允许极轻微背景变化，但不应抢内容

### 卡片与面板

- 使用轻边框和柔和圆角
- 降低厚重阴影
- 去掉过强玻璃效果
- 卡片更接近“安静的信息容器”

## 动画策略

动画采用“保守但不无聊”的策略。

### 第一阶段动效

- `hover`
  - 按钮轻微提起
  - 卡片边框和背景轻微变化
  - 链接颜色过渡

- `scroll`
  - 区块淡入上移
  - 项目卡片顺序出现
  - 导航高亮切换

- `background`
  - 轻微背景明暗 / 渐变位移

- `theme transition`
  - 明暗主题切换保持柔和过渡

### 暂不实现

- 强 3D 动画
- 大幅翻转
- 粒子特效
- 高强度视差

### 预留接口

为了后续继续增强动画，保留：

- `data-section`
- `data-motion`
- `data-animate`
- 模块化滚动观察逻辑

## 响应式设计

必须明确适配三个层级：

### 手机

- 单列布局
- 导航简化
- Hero 内容上下排布
- 按钮纵向或紧凑排列

### 平板

- 过渡到两列结构
- 保持足够留白

### 桌面

- Hero 区有更明确的视觉节奏
- 项目和信息卡片具备稳定网格系统

## 标签页与 favicon

### 标题

- 页面 `<title>` 改为 `secularism`

### favicon

- 替换旧的默认图标
- 使用简洁、可辨识、缩小后清晰的图标
- 风格建议为单色、几何、字母化 mark

## 实现边界

本轮重构范围：

- 重构现有首页
- 切换到 Tailwind 样式体系
- 重做视觉风格
- 保留主题切换
- 保留并优化轻量动画

本轮不做：

- 多页面博客
- 重型 UI 库整站接管
- 复杂表单后端
- 夸张动效系统

## 完成标准

- 首页比当前明显更朴素、更干净
- 页面标题为 `secularism`
- favicon 已替换
- Tailwind 成为主样式方案
- 不同屏幕尺寸下布局稳定
- 动画存在但不过度
- GitHub Pages 可正常部署

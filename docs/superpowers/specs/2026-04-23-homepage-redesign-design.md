# GitHub Pages 个人主页重构设计

**日期：** 2026-04-23

## 目标

将当前仓库重构为一个适合放在简历中的单页个人主页。主页需要兼顾“极简未来感”和“招聘场景可快速阅读”，同时保留后续加入滚动动画、卡片翻转、视差等视觉增强效果的扩展口子。

## 设计原则

- 单页滚动，不拆分多页面
- 顶部固定锚点导航，支持快速跳读
- 默认暗色光晕主题，同时支持切换到浅色玻璃主题
- 内容先使用 mock 数据，后续可直接替换
- 结构先稳定，动效后续渐进增强
- 保持静态部署友好，适合 GitHub Pages

## 信息架构

首页按以下区块组织：

1. `Hero`
   - 名字 / 角色 / 一句话定位
   - 主按钮：查看项目、联系我
   - 次按钮：GitHub / 简历入口
   - 右上角：导航、主题切换

2. `About`
   - 简短自我介绍
   - 个人标签
   - 一段带个人气质的说明文字

3. `Projects`
   - 3 个重点项目卡片
   - 每个卡片包含简介、技术栈、亮点、链接

4. `Stack`
   - 按能力分组展示，而非技能词云
   - 分为前端、工程化、后端基础、协作工具

5. `Contact`
   - 邮箱、GitHub、博客等
   - 明确的联系引导语

## 视觉方案

### 暗色主题

- 深色底 + 柔和蓝紫光晕
- 强调高级感与未来感
- 卡片采用低对比玻璃质感

### 浅色主题

- 冷白 / 雾灰 / 淡蓝色系
- 更适合简历阅读和日间浏览
- 保持与暗色主题一致的布局节奏

### 主题切换

- 页面顶部提供主题切换按钮
- 使用 `localStorage` 记忆用户选择
- 首次访问可参考系统主题，但站点默认使用暗色主题
- 切换时保留平滑过渡能力

## 交互设计

- 顶部导航点击后平滑滚动到目标锚点
- 当前阅读区块在导航中高亮
- 卡片与按钮支持轻量悬停反馈
- 保留滚动动画扩展口子，但首版不实现重动效

## 动画扩展预留

为了方便后续加入滚动条驱动动画、卡片翻转、分段显现等能力，首版结构需要提前预留以下扩展点：

- 每个区块设置统一的 `data-section` 和 `data-animate` 标识
- 每个卡片设置可扩展的 `data-motion` 标识
- JavaScript 侧保留动画配置入口，允许后续引入 Intersection Observer 或时间轴库
- 样式层使用 CSS 变量和状态类，避免未来动画改造时重写结构

## 技术方案

首版采用原生静态站点结构重建，而不是直接引入 Vue 工程。原因如下：

- 当前仓库本质是 GitHub Pages 静态发布仓库
- 直接重建静态首页可立即部署，无需等待依赖安装和构建链路稳定
- 页面规模较小，原生模块化 HTML/CSS/JS 足以支撑首版
- 后续如需要增强为 Vue 或引入构建工具，可在保留设计结构的前提下平滑迁移

## 文件组织原则

- `index.html`：页面骨架与语义化区块
- `styles/`：主题变量、布局、组件样式
- `scripts/`：主题、导航、数据、交互控制
- `tests/`：纯逻辑测试，优先覆盖主题解析与区块配置

## 非目标

首版不包含以下内容：

- 多页面博客系统
- 复杂 3D 场景或重粒子特效
- 后端表单提交系统
- CMS 或内容管理后台

## 完成标准

- 首页可直接在 GitHub Pages 静态托管
- 亮暗主题可切换且视觉一致
- 导航锚点滚动和区块高亮可用
- 内容结构完整且 mock 数据可替换
- 动画扩展点已预留，后续可渐进增强
